# CSM 系统启动说明

## 问题诊断

如果无法打开 http://localhost:3000，请按以下步骤检查：

### 1. 检查 Node.js 是否安装

在终端运行：
```bash
node -v
npm -v
```

如果没有安装 Node.js，请：
- **macOS**: 使用 Homebrew 安装：`brew install node`
- **或访问**: https://nodejs.org/ 下载安装

### 2. 安装项目依赖

在项目根目录运行：
```bash
# 安装根目录依赖
npm install

# 安装服务器依赖
cd server
npm install
cd ..

# 安装客户端依赖
cd client
npm install
cd ..
```

### 3. 启动项目

**方法一：使用启动脚本（推荐）**
```bash
chmod +x start.sh
./start.sh
```

**方法二：手动启动**
```bash
# 在根目录运行（同时启动前后端）
npm run dev

# 或分别启动
npm run dev:server  # 后端：http://localhost:3001
npm run dev:client  # 前端：http://localhost:3000
```

### 4. 常见问题

#### 问题1: 端口被占用
如果 3000 或 3001 端口被占用，可以：
- 修改 `client/vite.config.ts` 中的端口号
- 修改 `server/src/index.ts` 中的端口号
- 或关闭占用端口的程序

#### 问题2: 依赖安装失败
```bash
# 清除缓存后重新安装
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

#### 问题3: 权限问题
```bash
# 给启动脚本添加执行权限
chmod +x start.sh
```

### 5. 验证服务是否启动

启动后，你应该看到：
- 前端服务运行在：http://localhost:3000
- 后端服务运行在：http://localhost:3001

在浏览器访问 http://localhost:3000，应该能看到登录页面。

### 6. 检查日志

如果启动失败，查看终端输出的错误信息：
- 红色错误信息会提示具体问题
- 检查是否有端口冲突
- 检查依赖是否完整安装
