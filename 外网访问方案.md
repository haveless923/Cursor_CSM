# CSM 系统外网访问方案

如果你的同事在不同的网络环境下（比如在家办公、不同办公室等），需要通过外网访问系统，有以下几种方案：

## 方案一：内网穿透（推荐，最简单快速）⭐

使用内网穿透工具，让外网可以访问你的本地服务器。

### 使用 ngrok（推荐）

1. **安装 ngrok**：
   ```bash
   # macOS
   brew install ngrok
   
   # 或访问 https://ngrok.com/ 下载
   ```

2. **注册账号**（免费）：
   - 访问 https://dashboard.ngrok.com/signup
   - 注册后获取 authtoken

3. **配置 ngrok**：
   ```bash
   ngrok config add-authtoken 你的token
   ```

4. **启动内网穿透**：
   ```bash
   # 同时穿透前端和后端
   ngrok http 3000
   ```

5. **获取访问地址**：
   - ngrok 会显示一个公网地址，例如：`https://xxxx.ngrok.io`
   - 将这个地址告诉同事

6. **修改前端配置**（需要修改API地址）：
   - 由于前端和后端分离，需要配置代理或修改API地址

### 使用 Cloudflare Tunnel（免费，无需公网IP）

1. **安装 cloudflared**：
   ```bash
   brew install cloudflared
   ```

2. **启动隧道**：
   ```bash
   cloudflared tunnel --url http://localhost:3000
   ```

3. **获取访问地址**：
   - 会显示一个 `*.trycloudflare.com` 的地址
   - 将这个地址告诉同事

## 方案二：部署到云服务器（长期方案）

将系统部署到云服务器（阿里云、腾讯云、AWS等），这样所有人都可以通过公网IP访问。

### 部署步骤

1. **购买云服务器**
2. **在服务器上安装 Node.js**
3. **上传代码到服务器**
4. **配置域名和SSL证书**（可选）
5. **启动服务**

## 方案三：使用 VPN

如果公司有VPN，可以让同事通过VPN连接到公司网络，然后访问你的内网IP。

## 方案四：修改为支持外网访问的配置

如果使用内网穿透，需要确保：
1. 前端可以访问后端API
2. 配置正确的CORS
3. 使用相对路径或配置代理
